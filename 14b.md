# ObraSync — Resumo Completo do Sistema

> Documento gerado em 2026-06-10. Versão do app: `v1.8.0` (2026-06-10, conforme `app.js`; o `README.md` ainda menciona `v1.7.0` de 2026-06-08).

## 1. Visão Geral

O **ObraSync** é um sistema web de gestão integrada para empresa de engenharia, cobrindo **obras/projetos, financeiro, comercial e contabilidade gerencial**. Roda em produção em `https://schimanskiengenharia.com.br/financeiro`, com frontend em `/var/www/financeiro`, dados persistentes em banco MariaDB/MySQL e arquivos de dados fora da pasta pública (`/var/lib/financeiro`).

## 2. Tecnologias

| Camada | Tecnologia |
|---|---|
| Frontend | HTML, CSS e JavaScript puro (sem frameworks, sem build) |
| Backend | PHP (API única em `api/index.php`, ~2.065 linhas) |
| Banco de dados | MariaDB/MySQL, charset `utf8mb4` |
| Servidor | Apache (com `.htaccess` na raiz e em `api/`) |
| Configuração | `/etc/financeiro/config.php` (fora da pasta pública; exemplo em `api/config.sample.php`) |
| Deploy | `deploy.php` + cron de backup (`api/cron/jobs.php`) |

Estrutura recomendada em produção:

- `/var/lib/financeiro` — `backups/` e `uploads/` (fora da pasta pública).
- `/etc/financeiro/config.php` — credenciais do banco (nunca sobrescrever em atualizações).

## 4. Módulos do Sistema (~100 módulos no frontend)

A navegação é organizada em 7 seções na sidebar:

### Dashboard
- KPIs, gráficos e indicadores gerais (Dashboard ObraSync).

### Cadastros
- Clientes, Fornecedores, Produtos, Serviços, Categorias financeiras, Centros de custo, Contas bancárias.

### Obras/Projetos (eixo central do sistema)
- Obras/Projetos, Custos por obra, Receitas por obra.
- **Orçamentos de Obras** com base **SINAPI 04/2026 (UF padrão MS)**: referências, insumos, composições, itens de composição, mão de obra, famílias/coeficientes, manutenções, importador XLSX/CSV.
- Composições próprias, Cotações, **Curva ABC**.
- **Cronograma físico-financeiro**, marcos da obra, **Gantt simplificado**, integração inicial com **Microsoft Project (XML)**.
- Notificações da obra, links de acompanhamento, pedidos de compra, relatórios técnicos.
- **Notas Fiscais / Documentos Fiscais** (upload, download, vínculo com obra).
- **Agenda** e **Kanban** integrados.
- Relatório por obra.

### Financeiro
- Contas a receber, Contas a pagar, Movimentações de caixa, Fluxo de caixa, Conciliação bancária.

### Comercial
- Orçamentos, **Propostas comerciais** (itens, histórico de status, arquivos, vínculos proposta-orçamento, variáveis dinâmicas).
- **Gerador de proposta a partir de orçamento de obra**, com modelos editáveis, pré-visualização e impressão/PDF A4.
- Taxonomia: Áreas/Disciplinas → Tipos de atuação → Subtipos/Serviços.
- Vendas/Contratos.

### Contabilidade Gerencial
- Plano de contas, Lançamentos contábeis, **DRE gerencial**, Documentos fiscais, Impostos.

### Relatórios
- Relatório financeiro, por cliente, por fornecedor, por centro de custo, por obra/projeto; Exportações.

### Configurações
- Dados da empresa, Usuários, **Permissões/Perfis (RBAC por módulo e ação)**, Versão do sistema.
- **Estruturas editáveis pelo administrador**: tipos de obra, status de obra, etapas padrão, marcos padrão, campos personalizados, modelos de relatório, tipos de documento, checklists, tipos de medição, formas de pagamento, mensagens padrão (WhatsApp), regras de visualização.
- Configuração SINAPI, Backup local, Preferências, Migração do localStorage para o banco, Log de auditoria, Meu Perfil.

## 6. API PHP (`api/index.php`)

- Endpoint único com roteamento por segmentos de URL e mapa de recursos (`resource_map()`), gerando **CRUD genérico** (list/get/create/update/delete) para todas as entidades.
- Rotas especiais:
  - `bootstrap` — carga inicial de todos os dados.
  - `login` — autenticação (POST).
  - `backup` — exportação/importação de backup.
  - `migrate` — migração do payload do `localStorage` para o banco.
  - `sinapi-upload` / `sinapi-import` — upload e importação da base SINAPI (CSV/TXT/XLSX), com validação de extensão e MIME type.
  - `project-upload` — upload de XML do Microsoft Project.
  - `module=agenda` — endpoints dedicados da agenda.
- Automações transacionais em mudanças de status (`obra_cronograma_marcos`, `purchase_orders`).
- Headers de segurança (`X-Content-Type-Options: nosniff`), respostas JSON padronizadas.

## 7. Frontend (`app.js`)

- SPA monolítica em JS puro: estado em memória + `localStorage` (`finconta.v1`) com sincronização via API.
- **Autenticação com sessão local**: timeout de inatividade de 30 min, aviso 5 min antes, `API_BASE` absoluto em produção.
- **RBAC no cliente**: perfis (admin, financeiro, comercial, etc.), visibilidade de módulos e permissão de edição por módulo.
- Helpers de formatação pt-BR: moeda (BRL), datas, máscaras de telefone/CPF/CNPJ/CEP, validação de CPF/CNPJ, percentuais.
- Sidebar com seções recolhíveis, favoritos, filtros globais por data/cliente/projeto.
- Responsividade para celular, tablet, notebook, desktop e telas grandes.

## 8. Segurança (implementada)

- Autenticação real na API (login, sessão, sem bypass em produção — bypass `AUTH_BYPASS_FOR_TESTS` restrito a desenvolvimento).
- Configuração sensível fora da pasta pública (`/etc/financeiro/config.php`).
- Uploads validados (extensão + MIME) e armazenados fora da web root.
- Backup manual e automático via cron, com instruções de restauração no README.
- Commit `466f337` implementou um pacote de 10 itens de segurança.
